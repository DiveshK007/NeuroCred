## Getting Started

### New to NeuroCred?

1. Start with the [Architecture Overview](architecture.md) to understand the system
2. Read the [Developer Setup Guide](developer/setup.md) to set up your environment
3. Check out the [API Documentation](API.md) to understand the API
4. Review the [Integration Guide](INTEGRATION.md) to integrate NeuroCred

### Deploying to Production?

1. Read the [Deployment Guide](DEPLOYMENT.md)
2. Review [Security Documentation](SECURITY.md)
3. Set up [Monitoring](MONITORING.md) and [Alerting](ALERTING.md)
4. Configure [Backup Strategy](BACKUP_STRATEGY.md)

### Integrating NeuroCred?

1. Start with the [Integration Guide](INTEGRATION.md)
2. Review [API Documentation](API.md) for API endpoints
3. Check [Smart Contract Documentation](CONTRACTS.md) for on-chain integration
4. Review [Security Best Practices](SECURITY.md#security-best-practices)

## Documentation Standards

All documentation follows these standards:

- **Markdown Format**: All docs are in Markdown
- **Code Examples**: Include working code examples
- **Diagrams**: Use Mermaid for diagrams
- **Versioning**: Document version compatibility
- **Links**: Use relative links within docs

## Contributing to Documentation

When contributing to documentation:

1. Follow the existing structure and style
2. Include code examples where applicable
3. Update the table of contents if adding new sections
4. Test all code examples
5. Use clear, concise language
